# 登录问题排查指南

## 问题：无法登录

### 可能原因和解决方法

#### 1. 数据库连接失败

**检查步骤：**
1. 确认 MySQL 服务是否运行
   - Windows: 打开服务管理器，查找 MySQL 服务，确保状态为"正在运行"
   - 或使用命令行：`net start MySQL`

2. 检查数据库连接信息
   - 打开 `dbmanager.cpp` 文件
   - 确认以下配置是否正确：
     - 主机地址（默认：127.0.0.1）
     - 端口（默认：3306）
     - 数据库名（默认：stock_trader）
     - 用户名（默认：root）
     - 密码（默认：HBsc75820306@）

3. 如果配置不同，可以通过环境变量设置：
   - `DB_HOST`: 数据库主机
   - `DB_PORT`: 数据库端口
   - `DB_NAME`: 数据库名
   - `DB_USER`: 用户名
   - `DB_PASSWORD`: 密码

#### 2. 数据库不存在

**解决方法：**
1. 连接到 MySQL
2. 创建数据库：
   ```sql
   CREATE DATABASE IF NOT EXISTS stock_trader;
   ```
3. 应用会自动创建所需的表结构

#### 3. MySQL 驱动未安装

**检查方法：**
1. 运行 Qt 安装目录下的 `qtdiag.exe` 查看已安装的数据库驱动
2. 或者检查 Qt 安装目录：`Qt/6.x.x/plugins/sqldrivers/` 是否有 `qsqlmysql.dll`

**如果驱动不存在：**
- 需要重新编译 MySQL 驱动，或使用包含 MySQL 驱动的 Qt 版本

#### 4. 用户未注册

**解决方法：**
1. 在登录界面点击"注册"按钮
2. 输入用户名（至少 3 个字符）和密码（至少 6 个字符）
3. 注册成功后，使用注册的用户名和密码登录

#### 5. 密码错误

**解决方法：**
- 确保输入的密码与注册时使用的密码完全一致
- 如果忘记密码，需要重新注册用户（目前不支持密码重置）

## 调试方法

### 查看错误日志
1. 运行应用时查看控制台输出
2. 查找以下关键词的错误信息：
   - "Database initialization failed"
   - "DB open failed"
   - "Login query error"

### 测试数据库连接
可以使用以下 MySQL 命令测试连接：
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p
```
输入密码后，如果连接成功，说明数据库服务正常。

### 检查数据库表
连接数据库后，检查表是否存在：
```sql
USE stock_trader;
SHOW TABLES;
```
应该能看到：users, orders, positions, trades, companies 等表。

## 常见错误信息

- **"数据库连接失败，请检查MySQL服务是否运行"**
  - MySQL 服务未运行或连接信息错误

- **"用户名或密码错误，请先注册"**
  - 用户不存在，需要先注册

- **"数据库查询错误"**
  - SQL 语法错误或表结构问题，检查数据库日志
